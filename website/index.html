UncoreBleed

AEX-free, High-Resolution, and Low-Noise Side-Channel Attacks on SGX Enclaved Execution.

UncoreBleed shows that active uncore PMCs can expose enclave-correlated memory behavior even when sensitive core PMCs are suppressed. The attack can recover secrets from production-mode SGX enclaves, including RSA private keys protected by TLBlur with AEX-Notify.

How it works

UncoreBleed turns a shared processor monitor into a precise enclave side channel.

The attack relies on an address-filterable uncore event in the mesh-to-memory subsystem. Instead of interrupting the enclave, it observes memory traffic that remains visible outside the core PMC suppression boundary.

AEX-free

No page faults, interrupts, or asynchronous enclave exits are needed to collect the signal.

High-resolution

M2M PKT_MATCH enables selected address monitoring at 64 B granularity, exposing cache-line scale behavior.

Low-noise

Hardware address filtering counts targeted memory blocks and suppresses unrelated system activity.

Demo

Recovering TLBlur-protected RSA keys.

TLBlur with AEX-Notify is designed around attacks that observe chosen execution points through asynchronous enclave exits. UncoreBleed changes the setting: it passively monitors selected DRAM accesses through uncore PMCs and recovers RSA private keys from a single decryption without triggering AEXs.

Questions & Answers

FAQ

What is the impact of UncoreBleed?

UncoreBleed shows that enclave secrets can remain exposed through uncore performance monitoring even when SGX suppresses sensitive core PMCs. The paper demonstrates picture reconstruction and TLBlur-protected RSA key recovery; this site focuses on the RSA case.

Who is the attacker?

The attack is relevant to a privileged adversary, such as compromised system software, monitoring SGX enclave execution through architectural performance-monitoring interfaces.

What is responsible disclosure status?

Responsible disclosure details should be filled in according to the authors' final publication and vendor coordination timeline. Until then, the site avoids releasing exploit code or operational attack instructions.

Does UncoreBleed affect Intel TDX?

The current paper studies SGX-capable Xeon server platforms and SGX enclaves. Intel TDX has a different threat model and architecture; the site does not claim a TDX break without dedicated evaluation.

What are uncore PMCs?

Uncore PMCs monitor shared processor components outside individual cores, such as mesh, memory, and I/O subsystems. UncoreBleed focuses on the M2M PKT_MATCH event.
